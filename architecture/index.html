Architecture

The diagram below shows the architecture that I used to implement the travel project.

Below are the steps of a GET request that retrieves both the landing page and the pictures.

  1. The client sends a request to the ec2 instance IP address.
  2. Nginx receives the request and starts processing it.
  3. Nginx first returns the html landing page.
  4. As the page loads, a request to retrieve the pictures is proxied by Nginx to the backend (gunicorn).
  5. Gunicorn receives the request to get the pictures and asks dynamoDB.
  6. DynamoDb receives the request and returns the pictures.
  7. The landing page is finally populated with the pictures.

Components

  • programming languages and frameworks: python, flask, react, js, html
  • cloud: ec2, dynamoDB, s3
  • web servers: nginx, gunicorn
  • protocols: http (rest)

Background

I built the travel project out of a desire to learn more about non-relational databases and collect memories and pictures from my recent trips to Belgrade in a fun and personable way. It turned into a full stack project where I learned a good deal about dynamoDB and EC2, Nginx and Gunicorn. I also brielfy touched upon the concepts of proxy server and cors.
